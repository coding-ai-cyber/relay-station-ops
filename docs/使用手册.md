# Sub2API 经营管理系统使用手册

## 1. 系统入口

前端后台地址：

```text
http://127.0.0.1:5173
```

后端接口文档：

```text
http://127.0.0.1:8000/docs
```

默认管理员账号：

```text
用户名：admin
密码：<strong-password>
```

首次正式使用前，建议进入「用户管理」创建自己的管理员账号，并停用或修改默认账号密码。

## 2. 系统核心口径

系统不是单纯记账，而是判断采购是否真的值得继续买。

核心指标：

- 所有成本：所有采购支出，包括测试失败、废号、不可用账号。
- 真实成本：确认后续会继续采购，并且实际参与运营的有效成本。
- 利润：收入 - 所有成本。
- 真实利润：收入 - 真实成本。
- 测试损耗：所有成本 - 真实成本。

常用判断：

- 如果利润为负，说明按全部支出计算业务亏损。
- 如果真实利润为正，但利润为负，说明业务可运营，但测试损耗偏高。
- 如果测试损耗长期过高，需要降低试错采购，淘汰供应商。

## 3. 推荐使用流程

建议按以下顺序使用：

1. 录入供应商。
2. 录入采购单。
3. 上传采购凭证。
4. 一键生成资产。
5. 录入账号测评。
6. Finalize 测评批次。
7. 录入收入。
8. 查看首页和盈亏报表。
9. 根据供应商排行、账号类型排行、成本倍率建议决定是否继续采购。

## 4. 用户管理

入口：

```text
用户管理
```

可操作内容：

- 新增用户。
- 编辑用户。
- 修改密码。
- 修改角色。
- 启用或停用用户。

角色说明：

- admin：管理员，拥有全部权限。
- finance：财务，适合管理收入、成本、报表。
- purchaser：采购，适合管理供应商、采购、资产。
- tester：测评，适合管理账号测评和账号状态。
- viewer：只读用户。

## 5. 供应商管理

入口：

```text
供应商
```

用于维护号商、服务器厂商、代理池厂商、域名厂商、支付/工具服务商等。

建议填写：

- 供应商名称。
- 供应商类型。
- 联系方式。
- 登录地址。
- 登录账号。
- 登录密码/密钥。
- 国家/地区。
- 是否继续合作。
- 供应商状态。
- 备注。

敏感字段说明：

- 登录账号、登录密码/密钥会加密存储。
- 默认列表不直接显示敏感内容。
- 点击查看敏感字段时会写入审计日志。

供应商状态建议：

- 正常：可以继续采购。
- 观察中：质量不稳定，需要小批量测试。
- 暂停：暂时不再采购。
- 拉黑：不再合作。

## 6. 采购记录

入口：

```text
采购记录
```

每一笔采购都应该录入一条采购单。

采购类型：

- 账号。
- 服务器。
- 代理。
- 域名。
- 软件。
- 其他。

关键字段：

- 采购编号。
- 采购类型。
- 产品名称。
- 数量。
- 单价。
- 总价。
- 币种。
- 支付方式。
- 订单链接。
- 采购日期。
- 是否计入所有成本。
- 是否计入真实成本。
- 成本状态。
- 凭证截图/PDF。

凭证上传：

- 支持图片和 PDF。
- 单文件不超过 20MB。
- 上传后采购单会绑定凭证。
- 列表中可点击「查看」打开凭证。

一键生成资产：

采购类型为账号、服务器、代理时，可以点击「生成资产」。

生成规则：

- 账号采购：按采购数量生成账号资产。
- 服务器采购：生成服务器资产。
- 代理采购：生成代理池资产。

注意：

- 已生成过资产的采购单再次点击会跳过，避免重复生成。
- 生成出来的资产仍可在对应资产页面继续编辑。

## 7. 账号资产

入口：

```text
账号资产
```

用于维护买来的账号。

常用字段：

- 账号编号。
- 账号类型。
- 套餐类型。
- 登录账号。
- 登录密码。
- 当前状态。
- 是否参与运营。
- 是否计入真实成本。
- 成本单价。
- 可用天数。

账号状态：

- 待测试。
- 可用。
- 不可用。
- 风控。
- 封禁。
- 已退款。
- 已废弃。

CSV 导入：

点击「CSV 导入」可以批量导入账号。

CSV 表头建议：

```csv
account_no,account_type,plan_type,login_account,login_password,status,participate_operation,include_real_cost,cost_unit_price
ACC-001,OpenAI,Plus,user@example.com,password123,pending_test,false,false,10
```

说明：

- `account_no` 必填。
- `account_type` 必填。
- `login_password` 会加密保存。
- 布尔值建议写 `true` 或 `false`。

- 到期时间。

## 9. 代理池资产

入口：

```text
代理池资产
```

用于维护住宅、机房、移动、动态、静态代理。

常用字段：

- 代理类型。
- 地区。
- 数量/流量。
- 单价。
- 总价。
- 到期时间。
- 成功率。
- 延迟。
- 是否适合账号登录。
- 是否适合 API 调用。
- 是否继续采购。
- 是否计入真实成本。

使用建议：

- 登录账号用的代理，重点看稳定性、地区一致性、风控情况。
- API 调用代理，重点看成功率、延迟、成本。
- 成功率低或延迟高的代理池，建议标记为不继续采购。

## 10. 账号测评

入口：

```text
账号测评
```

这是采购决策的核心模块。

每次从号商买账号后，建议创建一个测评批次。

批次字段：

- 批次编号。
- 账号类型。
- 采购数量。
- 采购总价。
- 初测通过数量。
- 7 天可用数量。
- 30 天可用数量。
- 封号数量。
- 退款数量。
- 有效账号数量。
- 表面单价。
- 真实有效单价。
- 综合评分。
- 结论。

录入单账号测评：

点击「录入结果」，选择账号后填写：

- 是否可登录。
- 是否有风控。
- 是否能使用目标模型。
- 是否需要固定代理。
- 请求成功率。
- 平均响应质量。
- 可用天数。
- 是否封号。
- 是否退款。
- 人工评分。
- 测试结论。

Finalize：

测评批次完成后点击「Finalize」。

Finalize 会自动：

- 重新计算批次有效率。
- 计算真实有效单价。
- 生成综合评分和结论。
- 回写采购单成本状态。
- 回写采购单是否计入真实成本。
- 同步对应成本记录。
- 更新账号状态。

真实有效单价公式：

```text
真实有效单价 = 批次采购总价 / 最终有效账号数量
```

例子：

```text
采购 10 个账号，总价 100
最终有效 8 个
真实有效单价 = 100 / 8 = 12.5
```

## 12. 成本管理

入口：

```text
成本管理
```

成本来源有两类：

- 采购自动生成的成本。
- 手动录入的成本。

采购成本：

- 创建采购单时会自动生成成本记录。
- 修改采购单金额、日期、真实成本标记时，会同步成本记录。

手动成本：

可用于录入：

- 支付手续费。
- 软件工具。
- 人工成本。
- 退款损失。
- 测试损耗。
- 其他成本。

成本标记：

- 计入所有成本：用于完整现金支出口径。
- 计入真实成本：用于实际有效运营口径。

```text
仪表盘
```

首页展示：

- 本月收入。
- 本月所有成本。
- 本月真实成本。
- 本月利润。
- 本月真实利润。
- 测试损耗。
- 当前可用账号数量。
- 当前不可用账号数量。
- 供应商排行。
- 最近采购记录。

## 14. 盈亏报表

入口：

```text
盈亏报表
```

包含以下报表：

### 14.1 月度盈亏明细

按月份查看：

- 收入。
- 所有成本。
- 真实成本。
- 利润。
- 真实利润。
- 测试损耗。

支持导出 CSV。

### 14.2 稳定货源成本倍率建议

用于判断长期稳定货源应该按多少倍率采购或定价。

公式：

```text
建议倍率 = 基础毛利倍率 + 稳定风险缓冲 + 无效率损耗缓冲 + 评分风险缓冲
```

表格展示：

- 供应商。
- 批次数。
- 采购账号。
- 有效账号。
- 有效率。
- 平均评分。
- 真实有效成本。
- 建议倍率。
- 建议售价。
- 稳定等级。
- 原因。

稳定等级：

- 长期稳定。
- 观察中。
- 高风险。

使用建议：

- 长期稳定且有效率高的供应商，可以继续采购。
- 观察中的供应商只适合小批量测试。
- 高风险供应商不建议继续采购。

### 14.3 账号类型真实成本排行

按账号类型聚合：

- 批次数。
- 采购数量。
- 有效数量。
- 有效率。
- 所有成本。
- 有效面值成本。
- 测试损耗。
- 真实有效单价。
- 平均评分。

用途：

- 判断 OpenAI、Claude、Gemini、Grok、Codex 等账号哪类成本最低。
- 判断哪类账号损耗最高。

### 14.4 采购批次有效成本排行

按单个测评批次展示：

- 测评批次。
- 供应商。
- 账号类型。
- 采购数量。
- 有效数量。
- 有效率。
- 采购总价。
- 表面单价。
- 真实有效单价。
- 测试损耗。
- 评分。
- 结论。

用途：

- 找出最值得复购的批次。
- 找出损耗最高的批次。
- 追踪某个供应商某次采购是否异常。

## 15. 审计日志

入口：

```text
审计日志
```

会记录敏感操作，例如：

- 查看供应商密钥。
- 查看账号密码。
- 查看服务器 SSH 密钥。
- 上传文件。

建议：

- 定期检查敏感字段查看记录。
- 多人使用时，为每个人创建独立账号，不要共用 admin。

## 16. 日常操作建议

每天：

- 查看首页利润、真实利润、测试损耗。
- 查看最近异常账号。
- 查看 30 天内到期资产。

每次采购后：

- 录入采购单。
- 上传凭证。
- 生成资产。
- 创建测评批次。

测评完成后：

- 录入单账号测评。
- Finalize 批次。
- 查看真实有效单价。
- 更新供应商状态和是否继续合作。

每周：

- 查看供应商排行。
- 查看账号类型真实成本排行。
- 查看稳定货源成本倍率建议。
- 清理高风险供应商。

每月：

- 查看月度盈亏报表。
- 导出 CSV 对账。
- 复盘测试损耗。
- 决定下月采购预算和优先供应商。

## 17. 采购决策建议

推荐继续采购：

- 有效率高。
- 真实有效单价低。
- 30 天可用率高。
- 售后稳定。
- 平均评分高。
- 建议倍率合理。

谨慎采购：

- 有效率一般。
- 评分中等。
- 批次数不足。
- 偶发封号或风控。

不建议采购：

- 有效率低。
- 测试损耗高。
- 真实有效单价明显偏高。
- 封号严重。
- 售后差。

拉黑：

- 连续废号。
- 拒绝售后。
- 欺诈或货不对版。
- 真实有效成本远高于其他供应商。

## 18. 当前限制

目前系统已经支持手动经营闭环，但以下功能需要后续接入 sub2api 数据后才能完全自动化：

- 自动同步用户充值。
- 自动同步用户消费。
- 自动同步 key 使用情况。
- 自动同步模型消耗。
- 按账号类型精确分摊收入。
- 计算真正的账号类型利润。

在接入前，收入需要手动录入，账号类型报表主要统计真实成本、损耗和采购质量。
