# Sub2API 经营管理系统开发设计方案

## 1. 系统定位

本系统是 sub2api 中转业务的独立内部经营管理后台，覆盖供应商信息、账号采购、服务器采购、代理池采购、账号质量测评、成本统计、收入统计、真实利润分析和后续采购决策。

它的核心价值是把“采购支出”和“经营有效性”连接起来，而不是只做流水账。

## 3. 核心成本口径

### 所有成本

所有采购支出，包括测试失败、废号、不可用账号、无效采购。

```text
所有成本 = sum(计入所有成本的成本记录)
```

### 真实成本

确认后续会持续采购、实际参与运营的有效成本。

```text
真实成本 = sum(计入真实成本的成本记录)
```

### 利润

```text
利润 = 收入 - 所有成本
```

### 真实利润

```text
真实利润 = 收入 - 真实成本
```

### 测试损耗

```text
测试损耗 = 所有成本 - 真实成本
```

### 真实有效单价

```text
真实有效单价 = 批次采购总价 / 最终有效账号数量
```

如果最终有效账号数量为 0，则真实有效单价为空，并将该批次标记为无效或高风险。

- 是否退款

综合评分建议：

```text
综合评分 =
可用率 * 30%
+ 稳定性 * 25%
+ 性价比 * 20%
+ 售后 * 10%
+ 供货稳定性 * 10%
+ 人工评分 * 5%
```

结论建议：

- 推荐采购：综合评分 >= 85，且 30 天可用率 >= 80%
- 谨慎采购：综合评分 65-84，或 30 天可用率 50%-79%
- 不建议采购：综合评分 40-64，或 30 天可用率 20%-49%
- 拉黑：综合评分 < 40，或出现严重售后、欺诈、连续废号

## 7. 权限设计

| 角色 | 权限 |

| admin | 全部权限 |
| finance | 收入、成本、利润报表、采购金额 |

| tester | 账号测评、账号状态更新 |
| viewer | 只读查看，无敏感字段权限 |

敏感操作：

- 查看供应商密码/密钥
- 查看账号密码
- 查看 SSH 密钥
- 删除采购单
- 修改是否计入真实成本
- 修改测评最终结论
- 导出含敏感数据的文件

以上操作必须写入 audit_logs。

## 8. 敏感字段安全

- 使用 AES-256-GCM 做字段级加密
- 主密钥从环境变量读取：APP_FIELD_ENCRYPTION_KEY
- 数据库只保存密文
- 前端默认显示为 ********
- 点击查看时调用 reveal 接口
- 后端校验权限并写审计日志
- 明文只用于本次展示，不写入前端持久缓存

## 11. 开发优先级

1. 搭建后端 FastAPI、PostgreSQL、Alembic
2. 建立 users、suppliers、purchases、cost_items、audit_logs
3. 实现登录、权限和敏感字段加密
4. 实现供应商和采购管理
5. 实现账号资产和账号测评
6. 实现收入、成本、仪表盘统计
7. 实现月度盈亏报表
8. 补充服务器、代理池、文件上传
9. 对接 sub2api 自动同步

## 12. 关键风险

- 真实成本口径必须第一版就定清楚，否则后续报表会失真。
- 敏感字段必须后端加密，不能只做前端隐藏。
- 测评结论会影响采购决策，修改结论必须可审计。
- 账号、代理、服务器等资产必须绑定采购单，否则无法计算真实成本。
- 第一版不要急着自动化，先保证手动录入和统计链路可靠。
